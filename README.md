## OrderService (REST API)

Сервис заказов управляет заказами на постройку космических кораблей. 
Все заказы хранятся в базе PostgreSQL (через драйвер pgxpool и squirrel).
Сервис взаимодействует с `InventoryService` для расчёта цены заказа и с `PaymentService` для проведения оплаты.

## InventoryService (gRPC)

gRPC-сервис `InventoryService`, отвечающий за хранение и предоставление информации о деталях для сборки космических кораблей.
Все данные о деталях хранятся в MongoDB.

## PaymentService (gRPC)

Сервис оплаты заказов симулирует работу платёжного шлюза. Он не сохраняет состояние, а просто принимает команду на
оплату, генерирует UUID транзакции, печатает его в консоль и возвращает вызывающей стороне.

## Auth Service (gRPC)
Сервис аутентификации пользователей. Отвечает за регистрацию и логин. 
Данные пользователей хранятся в PostgreSQL с хэшированием паролей (bcrypt), а активные сессии кэшируются в Redis.
Предоставляет gRPC Interceptor (middleware) для сквозной проверки прав доступа в других сервисах.

## Notification Service (Kafka Consumer)
Асинхронный сервис-уведомитель. Работает в фоне, подписываясь на топики Kafka (order.paid, ship.assembled). 
При получении события формирует шаблонное сообщение и отправляет его пользователю через интеграцию с Telegram Bot API.
